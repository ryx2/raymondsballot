## Required checks

Before pushing to `main`, run `bun run build` and only push if it succeeds.

## Candidate portrait workflow

Candidate photos are managed in two layers: a source layer of full-resolution
JPGs from Wikimedia Commons, and a build-time WebP cache that is what the
site actually serves.

### Directory layout

```
public/candidates/
  ├── *.jpg                  ← source files (committed, do not delete)
  └── cache/
      └── *.webp             ← servable cache (committed, regenerated by script)
```

### Adding or updating a candidate's photo

1. Drop a 640×640-or-larger source image into `public/candidates/<slug>.jpg`
   (or `.jpeg` / `.png`). The slug must match the candidate's `slug` field
   in `data/candidates.ts`.
2. Set `imageUrl: "/candidates/cache/<slug>.webp"` on the candidate record.
   The cache file does not have to exist yet.
3. Run `bun run convert:images`. The script writes a 640×640 cover-fit
   WebP at quality 85 into `public/candidates/cache/`. It uses
   sharp's `position: "attention"` cropping so faces stay centered.
4. The conversion is incremental — files whose source mtime hasn't moved
   since the last cache mtime are skipped. To force a full rebuild,
   `rm -rf public/candidates/cache && bun run convert:images`.

### Build integration

`package.json` defines `prebuild: "bun run convert:images"`, so any
`bun run build` regenerates the WebP cache before Next.js compiles. The
WebP files are committed to the repo so Vercel can serve them on the
first deploy without needing sharp at build time, but the prebuild hook
keeps them in sync if a developer forgets to run the script manually.

### Why the cache (and not just <Image>)

`next/image` already handles WebP/AVIF transcoding on demand and caches
the optimized output under `.next/cache/images/`. We pre-generate the
WebP layer anyway because:

- The `.next/cache` lifetime is per-deployment; the in-repo cache
  survives across deployments and across `vercel build` runs.
- Pre-generated assets are served as plain static files at
  `/candidates/cache/<slug>.webp` — no image-optimization Lambda invocation,
  no `Image` component required for thumbnails.
- The cache is portable: any host (Vercel, Netlify, plain S3) serves it
  without configuration.

`<Image>` will still further optimize the WebP into AVIF where supported.

### Sharp version

The script uses sharp via Bun (transitive dependency through
`next` and `@tailwindcss/postcss`). If sharp is missing, run
`bun add -d sharp`. The `ignoreScripts` and `trustedDependencies`
entries in `package.json` allow sharp's native binaries to install
correctly under Bun.
